Implement r2d2-based connection pooling for SQLite backend to enable concurrent access and reduce connection overhead.

Purpose: Direct `Connection::open()` calls in `core.rs` prevent concurrent access and incur open/close overhead. Connection pooling enables multiple threads to access the database concurrently while reusing connections.

Output: `PoolManager` wrapper, updated `SqliteGraph` struct using pool, modified `connection()` method returning pooled connections.
</objective>

pub fn open<P: AsRef<Path>>(path: P) -> Result<Self, SqliteGraphError> {
    let conn = Connection::open(path).map_err(...)?;
    ensure_schema(&conn)?;
    Ok(Self::from_connection(conn))
}
