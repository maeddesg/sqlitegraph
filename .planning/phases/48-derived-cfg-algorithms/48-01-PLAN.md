# Key decisions from research
- Cytron et al. algorithm: walk up idom tree from each predecessor
- For each node n in reverse postorder: for each predecessor p of n: walk from p up idom tree to idom(n), adding n to each node's DF
- Iterated DF: fixed-point computation starting from definition nodes
- Entry node has idom[entry] = None, must handle this edge case
- Complexity: O(N^2) worst case for DF, O(N * iterations) for IDF

# DominatorResult API from Phase 47
- DominatorResult.dom: AHashMap<i64, AHashSet<i64>> - full dominance sets
- DominatorResult.idom: AHashMap<i64, Option<i64>> - immediate dominator tree
- DominatorResult.dominates(dominator, node): bool - convenience method

2. Result structs:
```rust
use ahash::AHashMap;
use ahash::AHashSet;

/// Dominance frontier result for a CFG
#[derive(Debug, Clone)]
pub struct DominanceFrontierResult {
    /// node -> set of nodes in its dominance frontier
    pub frontiers: AHashMap<i64, AHashSet<i64>>,
}

impl DominanceFrontierResult {
    pub fn frontier(&self, node: i64) -> Option<&AHashSet<i64>> {
        self.frontiers.get(&node)
    }

    pub fn in_frontier(&self, n: i64, m: i64) -> bool {
        self.frontier(n).map(|set| set.contains(&m)).unwrap_or(false)
    }
}

/// Iterated dominance frontier result (for SSA phi-placement)
#[derive(Debug, Clone)]
pub struct IteratedDominanceFrontierResult {
    /// Set of nodes needing phi-functions
    pub phi_nodes: AHashSet<i64>,
    /// Number of iterations to reach fixed point
    pub iterations: usize,
}
```

3. Core algorithm following Cytron et al.:
   - `compute_dominance_frontiers(graph, idom, all_nodes) -> AHashMap<i64, AHashSet<i64>>`
   - Algorithm: For each node n in reverse postorder (or any order):
       * For each predecessor p of n:
           - runner := p
           - while runner != idom(n):
               * DF(runner) := DF(runner) ∪ {n}
               * runner := idom(runner)
               * if runner is None: break (reached entry)
   - `iterated_dominance_frontier_internal(graph, df_result, definition_nodes) -> IteratedDominanceFrontierResult`
   - Algorithm: Start with definitions, compute DF, then iterate to fixed point

4. Public functions:
   - `dominance_frontiers(graph: &SqliteGraph, dom_result: &DominatorResult) -> Result<DominanceFrontierResult, SqliteGraphError>`
   - `dominance_frontiers_with_progress(graph: &SqliteGraph, dom_result: &DominatorResult, progress: impl ProgressCallback) -> Result<DominanceFrontierResult, SqliteGraphError>`
   - `iterated_dominance_frontiers(graph: &SqliteGraph, dom_result: &DominatorResult, definition_nodes: &AHashSet<i64>) -> Result<IteratedDominanceFrontierResult, SqliteGraphError>`

5. Error handling:
   - Handle empty predecessor sets (unreachable nodes)
   - Handle idom[entry] = None without infinite loop
   - Add iteration limit (100) for IDF to prevent non-termination

Do NOT use naive O(N^3) algorithm - use Cytron et al. O(N^2) walk-up algorithm as specified in research.
