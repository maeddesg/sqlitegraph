**Status:** Pending

## Summary

Wire automatic checkpointing into the WAL commit path. Currently, checkpoint triggering logic exists in `graph_integration.rs` but is NOT called from the main `commit_transaction()` path in `manager.rs`. This plan adds the missing integration and enables background checkpointing.

## Current State

**What Works:**
- `V2WALManager::requires_checkpoint()` (manager.rs:487-494) - evaluates size/transaction count
- `GraphWALIntegration` has `auto_checkpoint` flag and calls `force_checkpoint()` after ops
- `V2WALCheckpointManager` has full checkpoint execution logic

**What's Missing:**
1. `commit_transaction()` in manager.rs doesn't call checkpoint trigger
2. No background checkpoint thread
3. `estimate_wal_size()` uses writer metrics, not actual file size

## Files to Modify

1. `sqlitegraph/src/backend/native/v2/wal/manager.rs`
   - Add checkpoint trigger to `commit_transaction()`
   - Improve `estimate_wal_size()` to check actual file
   - Add background checkpoint thread option

2. `sqlitegraph/src/backend/native/v2/wal/mod.rs`
   - Add `background_checkpoint` config option

3. `sqlitegraph/src/backend/native/v2/wal/checkpoint/core.rs`

## Implementation

### Task 02-01a: Add Checkpoint Trigger to commit_transaction()

**File:** `manager.rs`

In `commit_transaction()` after line 383 (after triggering group commit), add:

```rust
// Check if checkpoint is needed after commit
if self.requires_checkpoint() {
    // Spawn background checkpoint to avoid blocking commit
    let manager = self.inner.clone();
    tokio::spawn(async move {
        if let Err(e) = manager.force_checkpoint().await {
            error_log!("Background checkpoint failed: {}", e);
        }
    });
}
```

### Task 02-01b: Improve WAL Size Estimation

**File:** `manager.rs`

Update `estimate_wal_size()` (line 588) to check actual file size:

```rust
fn estimate_wal_size(&self) -> u64 {
    // Check actual WAL file size if available
    if let Ok(metadata) = std::fs::metadata(&self.config.wal_path) {
        return metadata.len();
    }

    // Fallback to writer metrics
    self.writer.get_total_bytes_written()
}
```
