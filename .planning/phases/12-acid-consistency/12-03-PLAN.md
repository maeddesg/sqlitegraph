<objective>
Add pre-commit constraint validation to the transaction commit path.

Purpose: Before a transaction is committed and flushed to WAL, validate that all WAL records satisfy database constraints. This prevents invalid or corrupt data from being persisted. The TransactionValidator already exists (wal/recovery/validator.rs) and can validate individual records - we need to integrate it into the commit path.

Output: Transactions are validated before commit, with constraint violations causing abort.
</objective>

    ```rust
    async fn validate_pre_commit(&self, tx_id: TransactionId) -> NativeResult<()>
    ```

    Implementation:
    1. Get transaction context from self.transactions.read()
    2. Extract all WAL records from context.wal_records
    3. For each record, perform basic constraint validation:
       - NodeInsert: Verify node_id > 0, node_data not empty
       - NodeUpdate: Verify node_id matches in both old_data and new_data
       - ClusterCreate: Verify cluster_offset aligned to V2_CLUSTER_ALIGNMENT, cluster_size > 0
       - EdgeInsert/Delete/Update: Verify neighbor_id > 0
       - FreeSpaceAllocate/Deallocate: Verify block_size > 0, block_offset aligned
    4. Return NativeBackendError::InvalidParameter if any constraint fails
    5. Return Ok(()) if all records pass validation

    Note: This is a lightweight pre-commit check. The full TransactionValidator from recovery module can be used later if more comprehensive validation is needed.
