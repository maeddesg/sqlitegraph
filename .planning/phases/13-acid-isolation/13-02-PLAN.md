<objective>
Implement wait-for graph population during lock contention to enable deadlock detection.

Purpose: The DeadlockDetector has cycle detection (has_cycle_util) but the wait-for graph is never populated. When a transaction cannot acquire a lock because another transaction holds it, we must record this dependency in the wait-for graph so cycle detection can find deadlocks.

Output: Functional wait-for graph that tracks lock dependencies between transactions, enabling deadlock detection in 13-03.
</objective>

    /// Add a wait edge: tx_waiter is waiting for tx_holder to release a lock
    pub fn add_wait_edge(&self, tx_waiter: TransactionId, tx_holder: TransactionId) {
        let mut wait_for_graph = self.wait_for_graph.write();
        wait_for_graph
            .entry(tx_waiter)
            .or_insert_with(HashSet::new)
            .insert(tx_holder);
    }
