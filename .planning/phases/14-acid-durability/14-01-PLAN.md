<objective>
Wire transaction-count checkpoint trigger to use actual transaction counter instead of LSN delta

Purpose: LSN delta is an inaccurate proxy for transaction count because LSN increments per WAL record, not per transaction. A single multi-record transaction would cause early checkpoint triggering. This plan adds a dedicated transaction counter that resets after checkpoint.

Output: Transaction-count checkpoint strategy uses accurate counter that increments per committed transaction
</objective>
