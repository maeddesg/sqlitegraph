  <files>sqlitegraph/src/backend/native/v2/edge_cluster/cache.rs (new)</files>
  <action>
Create a new `cache.rs` module in the edge_cluster directory with:

1. `TraversalAwareCache` struct with:
   - `entries: LinkedHashMap<CacheKey, CacheEntry>`
   - `access_pattern: AccessPatternTracker`
   - `max_capacity: usize`

2. `CacheEntry` struct with:
   - `data: EdgeCluster`
   - `access_count: u32` (for LRU-K)
   - `last_access: Instant`
   - `traversal_score: f64` (increases on traversal accesses)

3. `AccessPatternTracker` to detect:
   - Sequential neighbor access (traversal pattern)
   - Random node access (lookup pattern)
   - High-degree node detection

4. LRU-K eviction policy:
   - Track K=2 most recent accesses
   - Prioritize entries with high traversal_score
   - Evict entries with lowest combined score

Use `parking_lot::RwLock` for thread-safe access. Do NOT use `std::collections::HashMap` - use ` LinkedHashMap` for LRU ordering.

WHY NOT BETrees: Research shows BETrees degrade graph read performance by 20-50% due to buffer overhead. Clustered adjacency with intelligent caching is superior.
