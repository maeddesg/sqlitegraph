  <action>
Create new file `sequential_cluster_reader.rs` with:

1. **Module-level documentation**: Explain the purpose (single-I/O sequential cluster reads for confirmed linear chains)

2. **MAX_CLUSTER_BUFFER_SIZE constant**: Set to 512KB (512 * 1024 bytes). This bounds memory usage and prevents unbounded growth. Document the rationale (sufficient for ~128 clusters of 4KB each).

3. **SequentialClusterReader struct**: Stateless struct (no fields needed) with associated methods.

4. **read_chain_clusters() method**:
   - Signature: `pub fn read_chain_clusters(graph_file: &mut GraphFile, cluster_offsets: &[(u64, u32)]) -> NativeResult<Vec<u8>>`
   - Validate cluster_offsets is not empty (return InvalidParameter error if empty)
   - Calculate total_size by summing all cluster_size values
   - Validate total_size <= MAX_CLUSTER_BUFFER_SIZE (return RecordTooLarge error if exceeded)
   - Calculate start_offset from first tuple's offset
   - Pre-allocate buffer: `let mut buffer = vec![0u8; total_size as usize];`
   - Single I/O call: `graph_file.read_bytes(start_offset, &mut buffer)?;`
   - Return Ok(buffer)

5. **extract_neighbors() method**:
   - Signature: `pub fn extract_neighbors(buffer: &[u8], cluster_index: usize, cluster_offsets: &[(u64, u32)]) -> NativeResult<Vec<NativeNodeId>>`
   - Validate cluster_index < cluster_offsets.len()
   - Calculate byte_offset by summing sizes of all preceding clusters
   - Extract cluster_bytes slice: `&buffer[byte_offset..byte_offset + cluster_size]`
   - Deserialize: `let cluster = EdgeCluster::deserialize(cluster_bytes)?;`
   - Extract neighbors: `cluster.iter_neighbors().map(|id| id as NativeNodeId).collect()`
   - Return Ok(neighbors)

6. **Unit tests** (in #[cfg(test)] mod):
   - `test_read_chain_clusters_empty_offsets_returns_error`: Empty slice returns InvalidParameter
   - `test_read_chain_clusters_single_cluster`: Read single cluster successfully
   - `test_read_chain_clusters_multiple_contiguous`: Read multiple clusters in one I/O
   - `test_read_chain_clusters_exceeds_max_size`: Returns RecordTooLarge when total > MAX
   - `test_extract_neighbors_first_cluster`: Extract from index 0
   - `test_extract_neighbors_middle_cluster`: Extract from index 1 with preceding clusters
   - `test_extract_neighbors_invalid_index`: Returns error for out-of-bounds index

Use existing error types: NativeBackendError::InvalidParameter, NativeBackendError::RecordTooLarge.
Follow code style from linear_detector.rs (module docs, struct docs, method docs).
