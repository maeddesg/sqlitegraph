# Research Summary (from 54-RESEARCH.md)
- petgraph's VF2 algorithm is de facto standard for subgraph isomorphism in Rust
- Use petgraph::algo::isomorphism::subgraph_isomorphisms_iter() for enumeration
- Use petgraph::algo::isomorphism::is_isomorphic_matching() for boolean check
- Major performance pitfall: O(N!) worst-case - always use bounds (max_matches, timeout)
- Semantic matching via node_match and edge_match closures enables pattern constraints

# Algorithm Pattern (from existing modules)
- All algo modules follow same structure: types/bounds first, then result structs, then main functions, then progress variants
- All _with_progress variants accept ProgressCallback and report阶段性 progress
- All bounds types have builder pattern (with_max_*, etc.)
- All result types have helper methods for common queries
- Tests are in shared tests.rs module with 3+ tests per function

# Key Technical Decisions
- Add petgraph as dev dependency (already in research as standard)
- Convert SqliteGraph to petgraph::DiGraph internally for VF2 processing
- Use SubgraphPatternBounds for max_matches, timeout, max_pattern_size limits
- Return SubgraphMatchResult with matches, patterns_found, computation_time_ms
- Node/edge matching via closures passed through from user code
