## Summary

Centralize debug output control by creating a `debug` feature flag that gates all `log::debug!`, `log::info!`, `log::warn!`, and `log::error!` macros. Currently 48 files have debug logging scattered without unified control.

## Approach

Instead of replacing every log call with `if cfg!(feature = "debug") { ... }`, we'll use conditional compilation to strip debug logs in release builds and provide a feature flag for development builds.

## Files to Create

1. `sqlitegraph/src/debug.rs` - Centralized debug macros (new module)

## Files to Modify

1. `sqlitegraph/Cargo.toml` - Add `debug` feature
2. `sqlitegraph/src/lib.rs` - Add `debug` module
3. 48 files with debug logging - Update log macro calls to use new debug macros

## Implementation

### Step 1: Create debug.rs module

```rust
//! Centralized debug logging with feature flag control

#[cfg(feature = "debug")]
macro_rules! debug_log {
    ($($arg:tt)*) => {
        log::debug!($($arg)*);
    };
}

#[cfg(not(feature = "debug"))]
macro_rules! debug_log {
    ($($arg:tt)*) => {
        // Debug logging disabled - compile to nothing
    };
}

#[cfg(feature = "debug")]
macro_rules! info_log {
    ($($arg:tt)*) => {
        log::info!($($arg)*);
    };
}

#[cfg(not(feature = "debug"))]
macro_rules! info_log {
    ($($arg:tt)*) => {
        // Info logging disabled - compile to nothing
    };
}

// Similar for warn_log, error_log
```

### Step 2: Add feature to Cargo.toml

```toml
[features]
default = []
debug = ["log/debug"]  # Optional: requires log crate debug feature
```

### Step 3: Replace log macros in source files

Find and replace systematically:
- `log::debug!` → `debug_log!`
- `log::info!` → `info_log!`
- Keep error/warn in release builds (use existing macros or create variants)
