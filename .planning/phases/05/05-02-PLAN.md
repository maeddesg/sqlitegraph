# Key Context
Plan 05-01 implemented metadata persistence. This plan adds vector persistence and index restoration.

**Approach**: Pragmatic vector storage + index rebuild (from docs):
- Store vectors in hnsw_vectors table (BLOB of f32 array)
- On load: read vectors from database, rebuild HNSW graph structure
- Trade-off: O(N log N) rebuild cost vs simpler implementation

Database tables already exist:
- hnsw_vectors (id, index_id, vector_data BLOB, metadata TEXT, timestamps)
- hnsw_layers (for future optimization - not used in initial implementation)
</context>
