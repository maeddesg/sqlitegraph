# Milestone v1.2: Benchmark Infrastructure

**Status:** ✅ SHIPPED 2026-01-21

## Overview

Fix broken benchmark harness for honest public performance numbers. Complete benchmark infrastructure repair including HNSW API fixes, Native V2 temp-file lifetime issues, and execution of full benchmark suite with documentation updates.

## Phases

### Phase 23: Benchmark Fixes

**Goal**: Fix broken benchmark harness (HNSW API, temp-file init)
**Depends on**: Phase 22
**Requirements**: BENCH-01, BENCH-02
**Plans**: 2 plans

Plans:
- [x] 23-01: Fix HNSW API mismatch in hnsw.rs benchmark
- [x] 23-02: Fix Native V2 temp-file lifetime issue

**Success Criteria:**
1. HNSW benchmarks compile with correct 2-argument API
2. Native V2 benchmarks run without temp-file errors
3. All benchmark functions execute successfully

**Details:**

HNSW API mismatch required updating `HnswIndex::new()` calls from single-argument form to two-argument form `(name, config)`. This was needed after Phase 15 introduced multi-index support.

Native V2 temp-file lifetime issue required `std::mem::forget(temp_dir)` pattern to prevent premature TempDir deletion before V2 backend async file operations completed. Also fixed CheckpointExecutor graph_path resolution bug and reduced benchmark sizes to fit V2's 8MB node region limit.

## Milestone Summary

**Key Decisions:**
- Use `std::mem::forget(temp_dir)` pattern for Native backend benchmarks to prevent premature deletion
- Use `config.graph_path` instead of deriving from `wal_path.with_extension('v2')` for V2 graph file resolution
- Reduce BENCHMARK_SIZES to [100, 500] to fit V2 backend's 8MB RESERVED_NODE_REGION_BYTES limit
- Present honest benchmark numbers with both strengths and weaknesses disclosed

**Patterns Established:**
- HnswIndex::new(name, config) pattern: name first, then config
- std::mem::forget for preventing TempDir deletion in async benchmarks
- Benchmark methodology must be documented with hardware, sizes, and cache state

**Issues Resolved:**
- HNSW API mismatch preventing benchmarks from compiling
- Native V2 temp-file lifetime causing "No such file or directory" errors during benchmark execution
- CheckpointExecutor graph path resolution bug when benchmarks used .db extension
- V2 backend node region limit exceeded by original benchmark sizes

**Issues Deferred:**
- Chain traversal performance regression (2-10x slower than SQLite) - identified for future optimization
- Benchmark CI integration (BENCH-05) - deferred to future milestone

**Technical Debt Incurred:**
- None significant - fixes were targeted and minimal
