## v1.13 Requirements

Requirements for minimal in-process pub/sub system with ID-only events and channel-based delivery.

### Pub/Sub System Requirements

- [x] **PS-01**: In-process only — No networking, IPC, or cross-process communication
- [x] **PS-02**: Events emitted on commit only — Emission happens after commit_lsn written, not on rollback
- [x] **PS-03**: ID-only events — Events carry identifiers (node_id, edge_id, key_hash, snapshot_id), no payload data
- [x] **PS-04**: Channel-based delivery — Uses Rust std::sync::mpsc for in-process message passing
- [x] **PS-05**: Best-effort delivery — No blocking on dropped receivers, no delivery guarantees
- [x] **PS-06**: Snapshot-aware — All events include snapshot_id (commit LSN) for version consistency
- [x] **PS-07**: Public API — GraphBackend trait has subscribe() and unsubscribe() methods
- [x] **PS-08**: Subscription filtering — Filter by event_type, node_ids, edge_ids, key_hashes
