# Milestone v1.4.0: Pub/Sub Enhancements

**Status:** Planning
**Target:** v1.4.0
**Start Date:** 2026-02-03

## Overview

Enhance the sqlitegraph pub/sub system with high-value, low-complexity features based on the analysis in `docs/PUBSUB_COMPARISON.md`. This milestone focuses on enabling pattern-based subscriptions and efficient KV prefix scanning for agent messaging use cases.

**User Exclusion:** IPC and networking features are explicitly OUT OF SCOPE per user request ("not IPC/NETWORKING sqlitegraph was not created for networks").

## Goals

1. **KV Prefix Scanning** - Enable efficient retrieval of all KV entries with a given prefix
2. **Query by Name/Kind** - Enable finding nodes by `kind` or `name` without full graph scan
3. **Pattern Subscription Filters** - Enable glob-style pattern matching for subscriptions

## Success Criteria

1. `kv_prefix_scan()` method returns all keys with given prefix at a snapshot
2. `query_nodes_by_kind()` method returns all nodes with matching kind
3. `query_nodes_by_name_pattern()` method returns nodes matching a glob pattern
4. `SubscriptionFilter` supports `kind_patterns` and `name_patterns` fields
5. Pattern matching uses glob syntax (`*` for wildcard, `?` for single char)
6. All features work with both SQLite and Native V2 backends
7. CLI commands added for new features
8. Documentation updated with examples
9. All tests pass with 0 warnings

## Estimated Complexity

- **KV Prefix Scanning**: Very Low (direct iteration over HashMap)
- **Query by Kind**: Low (add index on node.kind or iterate)
- **Query by Name Pattern**: Low (glob matching on node.name)
- **Pattern Filters**: Low (glob parsing and matching)
- **Total**: ~3-5 days implementation
