## End-to-End Flows

### 1. CFG Analysis Pipeline ✅

```
dominators → dominance_frontiers → natural_loops
```

**Use case:** Analyze control flow structure of a function
- User runs `dominators --entry ID` to compute dominance tree
- User runs `dominance-frontiers --entry ID` to compute frontiers (for SSA placement)
- User runs `natural-loops --entry ID` to detect loop structures

**Status:** All commands wired, compilation successful.

### 2. Control Dependence to Slicing ✅

```
post_dominators → control_dependence → backward_slice
```

**Use case:** Find what affects a specific statement
- User runs `post-dominators --exit ID` to compute post-dominance
- User runs `control-dependence --exit ID` to compute control dependence graph
- User runs `backward-slice --target ID` to find all nodes affecting target

**Status:** All commands wired, compilation successful.

### 3. Multi-Constraint Path Enumeration ✅

```
dominators + control_dependence + natural_loops → enumerate_paths_with_dominance
```

**Use case:** Enumerate feasible execution paths with pruning
- User runs `enumerate-paths-constrained --start ID --enable-dominance --enable-cd --enable-loops`
- Internally computes dominators, control dependence, and natural loops
- Returns pruned path set excluding impossible branches

**Status:** Command wired, compilation successful.
